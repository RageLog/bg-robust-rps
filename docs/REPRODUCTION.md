# Reproduction guide

This document collects the practical steps required to reproduce the
results reported in the accompanying paper (`paper/main.tex`).

## 1. Environment

The paper's results were produced under the following environment.
Pinning these versions reproduces the headline numbers within the
single-seed limitation discussed in the Limitations section.

| Component | Version |
|---|---|
| Python | 3.11 (Colab default) or 3.10 |
| TensorFlow | 2.16, with `[and-cuda]` extra |
| CUDA / cuDNN | 11.8 / 8.9 (Colab T4/A100/L4 runtime) |
| NumPy | 1.26 |
| pandas | 2.2 |
| scikit-learn | 1.5 |
| SciPy | 1.13 |
| OpenCV | 4.10 |
| Pillow | 10.3 |
| rembg | 2.0.55 (with `[gpu]`) |
| ONNX Runtime (GPU) | 1.18 |
| MediaPipe | 0.10.14 (legacy Solutions API; falls back to Tasks API ≥ 0.10.21) |
| matplotlib | 3.8 |
| seaborn | 0.13 |

`pip install -r requirements.txt` installs all of the above. The
single random seed is fixed at `42` (set in `src/config.py`) and is
propagated to the splitter, NumPy, TensorFlow, and the data-loader
workers in `src/cv_data_loader.py` and `src/train.py`.

## 2. Datasets

The repository does not redistribute any dataset. Obtain them from
their original sources.

| Dataset | Used as | Source |
|---|---|---|
| RPS-Kaggle (DrGFreeman) | Raw RPS gesture images | https://www.kaggle.com/datasets/drgfreeman/rockpaperscissors |
| RPS-Kaggle (frtgnn) | Raw RPS gesture images | https://www.kaggle.com/datasets/frtgnn/rock-paper-scissor |
| RPS-Kaggle (sanikamal) | Raw RPS gesture images | https://www.kaggle.com/datasets/sanikamal/rockpaper-scissors-dataset |
| RPS-Kaggle (alexandredj) | Raw RPS gesture images | https://www.kaggle.com/datasets/alexandredj/rockpaper-scissors-rps |
| Indoor Scenes | Background pool for compositing | https://www.kaggle.com/datasets/itsahmad/indoor-scenes-cvpr-2019 |
| HaGRID Sample 30k 384p | Held-out external evaluation | https://www.kaggle.com/datasets/innominate817/hagrid-sample-30k-384p |

Place the merged RPS images under `datasets/raw/{rock,paper,scissors,none}/`
and the background scenes under `datasets/backgrounds/`. The HaGRID
slice is staged separately under `datasets/external_test/` and is
never used during training, validation, or hyper-parameter selection.

## 3. Run the orchestrated pipeline

### Local

```bash
python run_all_experiments.py --run-mode missing --tune_strategy both
```

Stages executed by the orchestrator:

1. **Data preparation.** For every ablation mode (`NO_SYNTH`,
   `REMBG_ONLY`, `RANDBG`, `INDOOR`, `FULL`, `STYLE_TRANSFER`,
   `GAN`, `RATIO_1X`, `NO_SHIFT`, `NO_ALPHA`) the synthetic dataset
   is generated by `src/generate_data.py` and split by
   `src/split_data.py`.
2. **Main training grid.** `src/train.py` is launched per
   (backbone, ablation, fine-tuning) cell with image-grouped
   5-fold cross-validation. Backbones: EfficientNetV2B0, ResNet50,
   MobileNetV3Small, DenseNet121, VGG16.
3. **Classical baseline.** `src/baseline_mediapipe_svm.py` trains
   a MediaPipe Hands + RBF-SVM landmark baseline on the same
   four-class split.
4. **Head ablation.** `src/train.py` is rerun on the representative
   FULL/EfficientNetV2B0 cell with `head_strategy ∈ {standard,
   spatial_pooling, attention}`.
5. **Temporal smoothing.** `src/optimize_temporal_smoothing.py`
   runs a deployment-side N×θ grid search on the trained
   EfficientNetV2B0 cell.
6. **Segmentation quality.** `src/evaluate_segmentation.py` reports
   the pseudo-IoU and edge-contrast statistics of the rembg masks.
7. **Cross-dataset evaluation.** `src/evaluate_cross_dataset.py`
   evaluates every trained checkpoint zero-shot on the HaGRID slice.
8. **Grad-CAM extraction.** `src/gradcam_vis.py` produces the
   per-backbone, per-class activation maps used in Figure 7.
9. **Vector graphics.** `src/plot_vector_graphics.py` renders the
   per-fold confusion matrices and training history into vector
   PDFs.

### Hosted (Colab)

`train_on_colab.ipynb` runs the same pipeline. The Environmental
Setup block stages the dataset from Drive; the Automated Pipeline
cell runs all stages above. Trained checkpoints and reports are
backed up incrementally to `MyDrive/RPC_Colab/{models,reports}/`.

## 4. Aggregate the per-fold reports

After the pipeline completes, regenerate the seven aggregate tables
(Q1_Tables 1–7) used by the paper:

```bash
python regenerate_reports.py
```

These CSVs are also pre-shipped in `reports/aggregated/` for inspection
without rerunning the experiments.

## 5. Rebuild the paper's figures and tables

```bash
python paper/scripts/make_stats.py
python paper/scripts/make_tables.py
python paper/scripts/make_extra_tables.py
python paper/scripts/make_table15.py
python paper/scripts/make_figures.py
```

Regenerating the figures takes well under a minute on CPU; statistics
take a few seconds.

## 6. Verify image-grouped splitting

A standalone proof of the image-grouped splitting logic, that does
not require the synthetic dataset to be present, lives at
`paper/scripts/verify_grouping_logic.py`. It builds a simulated
filename pool from the raw dataset, applies the production grouping
rule, and asserts that the train/validation intersection of group
identifiers is empty for every fold. Sample run output is committed
in `reports/aggregated/grouping_proof.{md,json}`.

## 7. Compile the paper

```bash
cd paper
pdflatex -interaction=nonstopmode main.tex
bibtex main
pdflatex -interaction=nonstopmode main.tex
pdflatex -interaction=nonstopmode main.tex
```

Or upload the contents of `paper/` to Overleaf as a new project and
set `main.tex` as the main document.

## 8. Known limitations of this reproduction

- **Single-seed.** Standard deviations reported in the tables reflect
  fold-to-fold variability under one split realisation, not estimator
  variance under independent re-splits. A 3-seed × 5-fold rerun
  would lift the paired-Wilcoxon n=5 floor (~0.0625) below 0.05.
- **Incomplete fold for one cell.** DenseNet121 / RANDBG /
  standard / standard is reported on n=4 folds because fold-1 was
  unavailable in the locally retained artefact. The paper's headline
  numbers do not depend on this cell; details are in the Limitations
  section of the paper.
- **External landmark availability.** The auxiliary
  MediaPipe Hands + RBF-SVM baseline detects a hand in only 37.3 %
  of the external HaGRID slice, which dominates its 59.2 % external
  accuracy.
